# docs/

This directory is the home for design specs and the task backlog.

## Conventions

- **`backlog.md`** — flat checkbox task list. Use `- [ ]` for open items, `- [x]` for done, `- [~]` for dropped.
- **`specs/`** — design docs and brainstorming outputs, one file per feature/initiative.

## Workflow

1. New tasks go into `backlog.md` as `- [ ]` items.
2. When starting a spec, create `specs/<slug>.md` and link it from the backlog item.
3. Mark items `[x]` when merged, `[~]` when abandoned (with a brief note why).
